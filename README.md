# مشروع تيتان (TITAN) - نظام مراقبة السلامة وإدارة المنشآت الصناعية الذكي

مرحباً بك في وثيقة الشرح التفصيلية لمشروع **TITAN** المطور بلغة **Flutter** وإطار عمل **Riverpod**. يهدف هذا المستند إلى تقديم شرح شامل لهيكل المشروع، البنية البرمجية، تفاصيل الواجهات (UI/UX)، حالات الاستخدام (Use Cases)، وتجربة المستخدم.

---

## 1. نظرة عامة على المشروع (Project Overview)
**مشروع تيتان (TITAN)** هو نظام متكامل ومتقدم لإدارة السلامة الصناعية والمراقبة الفورية داخل المصانع والمنشآت الحساسة. يقوم التطبيق بالربط مع نماذج الذكاء الاصطناعي (AI) الخاصة بكاميرات المراقبة لتتبع ورصد التهديدات والمخالفات الأمنية والصحية في بيئة العمل لحظياً.

### أهداف التطبيق الأساسية:
* **مراقبة معدات الوقاية الشخصية (PPE)**: رصد العمال الذين لا يلتزمون بارتداء الخوذات (Helmet)، السترات الفوسفورية (Vest)، القفازات (Gloves)، النظارات (Glasses)، الكمامات (Mask)، أو أحذية السلامة (Boots).
* **إدارة البوابات والأسطول (Gate & Fleet Monitoring)**: فحص لوحات السيارات وتحديد السيارات المصرح بدخولها (Authorized) أو المحظورة (Blocked) لمنع الاختراقات الأمنية.
* **رصد تهديدات الحرائق (Fire & Thermal Detection)**: استلام إنذارات الحرارة المرتفعة ومؤشرات الحرائق لحظياً مع تحديد الكاميرا المسؤولة ونسبة الثقة لتفادي الكوارث.
* **رصد سرعات المركبات (Speed Violation)**: مراقبة سرعة العربيات داخل المنشأة ومخالفة السيارات التي تتجاوز الحد المسموح به.
* **التقارير والإحصائيات (Analytics & Reports)**: تحليل البيانات وعرض رسوم بيانية تفصيلية للمخالفات اليومية والأسبوعية لمساعدة الإدارة على اتخاذ القرارات.
* **إدارة الصلاحيات والمستخدمين**: لوحة تحكم كاملة للمشرف (Admin) لإضافة وتعديل وحذف المستخدمين والمركبات المصرح لها.

## 2. الهيكل البرمجي للمشروع (Project Structure & Architecture)

يتبع المشروع معيار **Feature-First Architecture** (البنية القائمة على الميزات)، وهو أفضل معمارية لتطبيقات الفلاتر الكبيرة حيث يتم تقسيم الكود إلى ميزات مستقلة تحتوي كل ميزة على طبقة البيانات وطبقة العرض الخاصة بها لتسهيل التطوير والصيانة.

### شجرة المجلدات الأساسية لـ `lib`:

```text
lib/
│
├── core/                         # النواة المشتركة للتطبيق (الأمان، الشبكات، الثيم)
│   ├── constants/                # الثوابت (عناوين الـ APIs)
│   ├── network/                  # إعدادات الاتصال بالخادم والـ Interceptors
│   ├── services/                 # الخدمات المحلية (حفظ البيانات والتواكن)
│   └── theme/                    # الهوية البصرية وألوان الهوية للبراند
│
├── features/                     # الميزات الوظيفية للتطبيق (كل ميزة في مجلد مستقل)
│   ├── admin/                    # لوحة التحكم وإدارة العمال والأسطول
│   ├── auth/                     # المصادقة (تسجيل الدخول، استعادة كلمة المرور)
│   ├── home/                     # الشاشة الرئيسية والـ Dashboard والإنذارات اللحظية
│   ├── monitoring/               # شاشات المراقبة اللحظية (بوابات، خرائط، خرائط، حرائق، PPE)
│   ├── reports/                  # التقارير الإحصائية والرسومات البيانية
│   └── settings/                 # إعدادات الحساب وتفضيلات المستخدم
│
├── shared/                       # المكونات والـ Widgets المشتركة بين الشاشات
│   └── widgets/                  # أزرار، لوحات أرقام سيارات، هياكل تحميل مؤقتة
│
└── main.dart                     # نقطة انطلاق التطبيق وتهيئة الإعدادات العامة
```

---

### تفاصيل محتويات مجلد النواة `lib/core/`

* **`constants/api_constants.dart`**: يحتوي على عناوين ورابط الـ API الأساسي (`http://72.62.186.142:8000/api`) وجميع مسارات الطلبات (Endpoints) مثل تسجيل الدخول، المستخدمين، الأسطول، وسجلات الحرائق ومعدات الوقاية.
* **`network/`**:
  * **`dio_provider.dart`**: يوفر كائن الاتصال `Dio` مهيأ بالكامل لاستخدامه في جلب البيانات.
  * **`auth_interceptor.dart`**: معالج يعترض كافة الطلبات لإرفاق رمز المصادقة (Bearer Token) تلقائياً في الـ Headers لضمان الأمان والتعرف على الجلسة.
* **`services/token_service.dart`**: مسؤول عن تخزين وحذف الـ Access Token وبيانات المستخدم محلياً على ذاكرة الموبايل بشكل آمن لضمان بقاء المستخدم مسجلاً للدخول.
* **`theme/brand_colors.dart`**: يحفظ الهوية اللونية الرسمية لتطبيق **TITAN**:
  * **اللون الكحلي الوقور (Navy)**: `0xFF1C2A5A`
  * **اللون الأزرق المتوسط (Medium Blue)**: `0xFF00407C`
  * **الأخضر الليموني الحيوي (Lime Green)**: `0xFF73AA2A`
  * **الأخضر الفاتح (Light Green)**: `0xFFA8C587`
  * تدرجات الألوان الحركية (Gradients) للواجهات الفاخرة.

### تفاصيل محتويات مجلد الميزات `lib/features/`

تم تقسيم كل ميزة داخل التطبيق برمجياً إلى مجلدين رئيسيين:
1. **`data/`**: يحتوي على الـ **Models** (النماذج البرمجية للبيانات) والـ **Repositories** (المسؤول عن الاتصال بالإنترنت والـ Caching).
2. **`presentation/`**: يحتوي على الـ **Screens** (الواجهات) والـ **Providers** (إدارة الحالة - State Management عبر Riverpod).

#### أ. ميزة المصادقة (`features/auth/`)
* **`data/models/user_model.dart`**: تحويل بيانات تسجيل الدخول وتفاصيل الجلسة من وإلى الـ JSON.
* **`data/repositories/auth_repository.dart`**: مسؤول عن إرسال طلبات الـ Login والـ Logout والـ Profile وحفظ الجلسة في الكاش المحلي لتقليل استهلاك الشبكة.
* **`presentation/providers/auth_provider.dart`**: إدارة حالة تسجيل الدخول وتحديث الواجهة بمجرد المصادقة.
* **`presentation/screens/login_screen.dart`**: شاشة تسجيل دخول عصرية، مجهزة بمؤشرات التحميل، وحماية كلمة المرور.

#### ب. ميزة لوحة التحكم الإدارية (`features/admin/`)
* **`data/repositories/user_management_repository.dart`**: تنفيذ العمليات الأربعة (CRUD) للمستخدمين والمركبات على الخادم مع إعداد نظام كاش ذكي مستقل لكل قائمة.
* **`presentation/screens/`**:
  * **قائمة المستخدمين وتفاصيلهم**: (`users_list_screen.dart` / `add_user_screen.dart` / `edit_user_screen.dart` / `user_details_screen.dart`).
  * **إدارة أسطول السيارات**: (`vehicles_list_screen.dart` / `add_vehicle_screen.dart` / `edit_vehicle_screen.dart` / `vehicle_details_screen.dart`).

#### ج. ميزة الشاشة الرئيسية (`features/home/`)
* **`presentation/screens/home_screen.dart`**: لوحة القيادة التفاعلية (Dashboard) التي تعرض إحصائيات سريعة للمخالفات، قائمة فورية لآخر الإنذارات (حرائق، مخالفات سرعة، بوابات، ملابس وقاية) وولوج سريع للوظائف.
* **`presentation/screens/alert_details_screen.dart`**: شاشة تعرض تفاصيل وصورة أي مخالفة بوابة أو مركبة مرصودة.
* **`presentation/screens/ppe_alert_details_screen.dart`**: شاشة مخصصة لعرض تفاصيل مخالفات معدات الوقاية للعمال بالاسم والصورة.

#### د. ميزة المراقبة اللحظية (`features/monitoring/`)
* **`presentation/screens/factory_map_screen.dart`**: خريطة المنشأة لمتابعة توزع العمال وأماكن الخطر.
* **`presentation/screens/gate_monitoring_screen.dart`**: عرض لحظي ومباشر لكاميرا البوابات لرصد حركات الدخول والخروج.
* **`presentation/screens/ppe_logs_screen.dart`**: لوحة الفلترة الكبرى لـ 6 فئات لمعدات الوقاية مدعومة ببيانات الصفحات والكاش الفوري.
* **`presentation/screens/restricted_areas_screen.dart`**: مراقبة المناطق المحظورة التي لا يسمح بدخول العمال غير المصرح لهم إليها.
* **`presentation/screens/fire_logs_screen.dart`**: قائمة المراقبة الحرارية وإنذارات الحرائق لحظة بلحظة مع فرز نسبة الثقة (Confidence).
* **`presentation/screens/fire_log_details_screen.dart`**: شاشة الإنذار الكامل للحرائق، تقدم خيار كتم الجرس (Mute) أو تعليم الحادثة كـ (Mark Handled) مع عارض صور تفاعلي مكبر.

#### هـ. ميزة التقارير الإحصائية (`features/reports/`)
* **`presentation/screens/reports_screen.dart`**: شاشة تعرض رسوم بيانية وتوزيع المخالفات وتوجهها الزمني لمساعدة المهندسين على تحسين بيئة السلامة بالموقع.

---

### تفاصيل مجلد المكونات المشتركة `lib/shared/`

* **`widgets/custom_nav_bar.dart`**: شريط التنقل السفلي الفاخر، يعتمد على تصميم الكابسولة العائمة (Floating Bubble) مع أنيميشن وتدرج لوني مميز عند الانتقال بين الشاشات.
* **`widgets/license_plate_widget.dart`**: ودجت مرسومة بشكل جميل وواضح تماثل لوحات السيارات وتنسق الحروف والأرقام العربية والإنجليزية لتقديم تجربة قراءة فائقة السرعة للأدمن.
* **`widgets/skeleton.dart`**: تأثير الـ Skeleton Loading اللطيف لعرض هياكل رمادية متحركة ومضيئة أثناء تحميل البيانات بدلاً من دوائر التحميل التقليدية المزعجة.

## 3. تصميم وتجربة المستخدم (UI/UX & User Experience)

تم تصميم واجهات تطبيق **TITAN** بعناية شديدة لتلائم ظروف العمل الصعبة في المواقع الصناعية التي تتطلب سرعة استجابة فائقة وقراءة واضحة للمعلومات تحت أي إضاءة.

### مبادئ التصميم المطبقة:
1. **عناصر بصرية مبهرة (Rich Aesthetics)**: الاعتماد على درجات كحلي وزرقاء داكنة تمنح إحساب الأمان والمراقبة، مع توظيف درجات الأحمر الصارخ لإنذارات الحرائق والأخضر الليموني للـ PPE.
2. **الأنيميشن السلس (Micro-Animations)**: استغلال الـ `AnimationController` لتقديم تأثيرات انزلاق الشاشات (Slide Transitions) وتلاشي القوائم التدريجي (Staggered Fade-in) لتقليل إحساس الانتظار لدى المستخدم وجعل التطبيق نابضاً بالحياة.
3. **مؤشرات السلامة والمرونة**: تصميم بطاقات المخالفات بزوايا دائرية واسعة (BorderRadius 24 و 28) لتقليل الضغط البصري وإبراز صور الحدث بوضوح.
4. **العناية الفائقة بأداء الشبكة والـ Caching**:
   - **الذاكرة المؤقتة المزدوجة (Dual-Tier Cache)**: يتم حفظ القوائم الكبرى محلياً في الرام عند تحميلها أول مرة، وعندما يضغط المستخدم لمشاهدة تفاصيل أي مخالفة، تُعرض البيانات فوراً من الكاش دون أي عملية لودينج أو طلب شبكة إضافي.
   - **تخزين الصور المتقدم (Cached Network Image)**: يتم تخزين صور الكاميرات والعمال على قرص الهاتف محلياً بمجرد تحميلها لأول مرة، لضمان تصفحها بدون إنترنت وتجنب تحميلها مراراً وتكراراً لتقليل استهلاك السيرفر وحزم الإنترنت.

## 5. الخلاصة وطريقة التشغيل البرمجية

التطبيق يمثل واجهة احترافية متكاملة لبيئة إنتاجية حية (Production-Ready).
* **إدارة الحالة**: تتم بكفاءة تامة باستخدام **Riverpod Code Generation**.
* **الاتصال بالـ API**: مأمن ومحمي بالكامل ويدعم الـ Caching.

### لتوليد الأكواد التلقائية في المستقبل عند التعديل على الموديلز:
```bash
dart run build_runner build --delete-conflicting-outputs
```
